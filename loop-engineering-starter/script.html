DEMO — THE COMMAND — 1:00–1:30

Here's the command:


/loop 2m /goal Check tasks.md and implement every pending task

That's it.

/loop 2m — Claude checks every 2 minutes.
/goal — it keeps working until the condition is met.
tasks.md — a plain file where you write what you want done.

EXPAND — WHERE TO TAKE IT — 3:00–4:00

That's the starter. Here's what people build from here:

A state file — so the loop remembers what it did yesterday and resumes instead of restarting.

A verifier — a test suite or type check that fails bad code automatically. Without one, the agent grades its own homework.

MCP connectors — GitHub, Slack, Linear. The loop opens the PR and pings you when it's done. You just review.

/schedule — move the loop to the cloud so it runs after you close the laptop.

One thing though — don't skip the verifier. A loop without a real check is just the agent agreeing with itself. The articles are right about that part.
